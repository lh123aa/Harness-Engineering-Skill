# 项目目标
一个快速检测 Markdown 文件中所有链接有效性的 CLI 工具，帮助文档维护者及时发现死链。

# 仓库结构
- /src : 源码目录
- /tests : 测试文件
- /docs : 项目文档

# 开发约束
- Python 3.9+
- CLI 工具，跨平台兼容（Win/Mac/Linux）
- 仅依赖 requests 和 rich（报告输出），其余用标准库

# 风险识别与应对
- **风险：大量链接时检测速度慢**
  - 概率：中（超过 1000 个链接时）
  - 影响：用户等待时间长，可能误判超时
  - 缓解策略：并发检测（asyncio/aiohttp）+ 可配置超时

- **风险：被目标网站屏蔽（频率过高）**
  - 概率：中
  - 影响：检测结果不准确（假阳性）
  - 缓解策略：可配置请求间隔；User-Agent 轮换；尊重 robots.txt

# 已知技术债务
- **妥协点：** 先实现同步 requests + 线程池并发，不做全异步
  - **后果：** 极端大量链接时性能不如 aiohttp
  - **清偿计划：** v2.0 迁移到 aiohttp 全异步方案
