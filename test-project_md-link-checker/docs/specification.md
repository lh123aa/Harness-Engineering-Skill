# 项目规格：Markdown 死链检测工具

## 概述
一个 CLI 工具，递归扫描 Markdown 文件中的超链接，通过 HTTP 请求验证每个链接的有效性，输出结构化报告。

## 功能详设

### P0 链接提取
- **交互流程：** 用户输入文件路径/目录 → 递归扫描 .md 文件 → 正则提取所有 `[text](url)` 链接 → 去重 → 输出链接列表
- **状态定义：** 待扫描 → 扫描中 → 提取完成 / 提取失败
- **输入/输出：** 输入文件/目录路径；输出链接列表（含所在文件和行号）
- **错误处理：** 文件无法读取→跳过并警告；无 .md 文件→提示；链接格式异常→记录并继续
- **边界场景：** 空文件→输出空列表；超大文件（>10MB）→流式读取；图片链接 `![alt](img.png)`→过滤

### P0 并发检测
- **交互流程：** 获取链接列表 → 并发发送 HEAD/GET 请求 → 按状态码分类（正常/重定向/失效/超时）→ 输出检测结果
- **状态定义：** 待检测 → 检测中 → 检测完成 / 中断
- **输入/输出：** 输入链接列表；输出每个链接的状态码、响应时间、最终 URL
- **错误处理：** 超时→标记为超时并重试 1 次；DNS 解析失败→标记为域名不存在；SSL 错误→标记为证书问题
- **边界场景：** 网络断开→暂停并提示；所有链接失效→汇总报告；并发数限制（默认 10，可配置）

### P0 报告输出
- **交互流程：** 检测完成 → 汇总统计 → 输出到终端和/或文件
- **状态定义：** 汇总中 → 输出完成
- **输入/输出：** 输入检测结果数据；输出终端表格报告 + 可选 JSON/CSV 文件
- **错误处理：** 输出文件无法写入→仅终端输出；无失效链接→绿色提示"全部正常"
- **边界场景：** 0 个链接被检测→提示无链接；全部失效→红色警告+详细列表

### P0 超时与重试
- **交互流程：** 用户通过 `--timeout` 和 `--retry` 参数控制
- **配置项：** 默认超时 10s；默认重试 1 次；可分别配置
- **行为：** 超时后等待重试间隔 → 重试 → 仍失败则标记为失效

### P0 Glob 模式匹配
- **交互流程：** 用户输入 `**/*.md` 等模式 → glob 匹配 → 展开为文件列表
- **输入/输出：** 输入 glob 模式；输出匹配文件列表
- **默认行为：** 无参数时默认 `**/*.md`
